# corpus/ — 학습 코퍼스 원본 관리

김짠부 롤링 구글독스에서 export한 스크립트 원본을 모아두는 폴더.
파싱 → `corpus_editions` / `corpus_components` 테이블로 ingest (tech.md §13.2).

## ⚠️ 보안 (반드시)
- **`corpus/raw/` 는 gitignore — 절대 커밋 안 됨.** (스크립트 원본 = 민감, CLAUDE.md 보안 규칙)
- 이 `README.md`(컨벤션 문서)만 커밋된다.

## 폴더 구조
```
corpus/
├── README.md        ← 이 파일 (커밋 O)
└── raw/             ← export 원본 (커밋 X · gitignore)
    ├── 260322_ISA.md
    ├── 260406_대출vs투자.md
    └── ...
```

## export 컨벤션
- **형식**: Markdown(`.md`) 권장 — 구글독스 `파일 → 다운로드 → Markdown`. (`.txt`도 가능, 서식은 날아감)
- **단위**: 편(탭)별 1파일 = 총 8편. 탭별 export가 안 되면 전체 1파일도 OK(편 구분자로 분리).
- **파일명**: `YYMMDD_제목.md` (예: `260322_ISA.md`)

## 원문 라벨 → DB 컴포넌트 매핑 (파서 인식 · 실제 export 기준)
원문 라벨은 굵게(`**라벨**`) 표기, 섹션은 긴 구분선(`—----`)으로 분리. 라벨은 **이모지·공백·`/뒷말` 제거 후 키워드 매칭**(느슨하게).

| 원문 라벨(실제) | DB `corpus_components.type` |
|---|---|
| `**제목**` | `title` |
| `**썸네일**` | `thumbnail_copy` |
| `**더보기란/고정댓글**` | `description` |
| `**스크립트**` 또는 `**🎬 스크립트**` | `script` (※ "대본" 아님) |

- **1파일 = 1편** (탭별 분리 export → 파일 내 편 구분자 `🟢 [날짜]` 없음). `edition_date`/`topic`은 아래 골든 v1 매핑·파일명으로 채움.
- 썸네일/제목의 `1. 2. 3.` 번호 = `variant_idx`(0/1/2), 썸네일은 `(작은 박스: …)` 동봉. 최종 선택 → `is_final`.
- 전부 🟢 done·info → `include_in_training=true`.
- **파싱 시 정리**: Google Docs Markdown 노이즈(`\!`·`\&` 이스케이프, `![👉🏻][image1]` 이미지 참조) 제거.
