김짠부 제작 동료 AI 시스템

콘텐츠 제작 5단계(주제 → 제목·썸네일 → 구성 → 리서치 → 대본)를 AI 크루가 보조·대행하고, 발행 성과로 학습해 김짠부의 사고를 확장한다. 핵심 원칙: 김짠부는 매 단계 '선택'만, 이유는 AI가 현재 방식에 근거해 설명한다.

전체 흐름 — 🤖 AI가 후보를 제안하고, 👤 김짠부는 선택만. 선택·승인이 곧 다음 학습의 입력이 된다.

크루 (하이브리드 에이전트)

  • 반장 — 총괄 PD/오케스트레이터. 단계 진행과 파이프라인을 지휘.
  • 촉이 — 주제. 시청자 신호를 읽어 주제를 발굴.
  • 훅이 — 제목·썸네일. 클릭을 만드는 문구를 설계.
  • 구다리 — 구성. 영상의 전개 전략과 목차를 짠다.
  • 셜록 — 리서치. 5단계로 사실을 교차검증하고 설명 자료를 만든다.
  • 짠펜 — 대본. 김짠부 말투로 초안을 쓰고 출처를 연결.
각 단계는 AI가 후보 N개 + 이유를 내고 → 김짠부가 선택·수정 → 발행 후 성과와 연결해 회고 → 좋은 패턴을 L3 지식으로 승격하는 같은 루프를 돈다.

기호이 메뉴얼 읽는 법

다이어그램과 카드에서 반복되는 기호의 뜻이다.

🤖 AI 모델 호출 (단계당 1회, 리서치만 다중) 👤 사람만 — 선택·승인·검수 ⚙️ 코드 로직 (AI 0회) 🛡️ 자동 가드 (베낌·표절·인젝션 방어) 📖 읽기 입력 💾 저장
L1 원천 데이터 (자막·댓글·검색결과) L2 구조화 산출물 (제안·사실·대본 조각) L3 학습 지식 (말투·문구·구성 스타일·인사이트)

① 주제무엇을 다룰까

크루: 촉이 · roleId topic_scout

동작 구조

📖 채널 댓글 키워드(L1) · 외부 신호(유튜브·뉴스·검색 동향) · 과거 영상 성과 · 승인된 주제 인사이트(L3)를 읽는다.

🤖 AI 1회 호출로 주제 후보를 만든다. 5단계 중 유일하게 웹 검색을 사용.

출력

주제 후보 3개(A/B/C). 같은 주제도 시청자 수준(입문~고급)으로 분화. 💾 stage_proposals(L2).

{ title, reason, evidence_ids[], audience_level, audience_need }
사용법

👤 김짠부가 후보 하나를 선택(필요하면 수정). 선택된 주제가 ②로 넘어간다.

선택 자체가 학습 입력 — 어떤 주제 감각이 통했는지 누적된다.

② 제목·썸네일클릭을 만든다

크루: 훅이 · roleId hook_maker + thumbnail_maker (코드상 별도 단계)

제목 (hook_maker)

동작 구조

📖 선택 주제 + 말투(L3) + 제목 스타일(L3, 채널 과거 제목에서 학습) + (옵션) 외부 고조회 제목.

🤖 AI 1회 또는 학습된 스켈레톤으로 로컬 생성. 🛡️ 베낌 가드·스타일 부합도 자동 검사.

출력

제목 3안. 낚시가 아니라 핵심을 담은 직설. 💾 stage_proposals.

{ title, reason, evidence_ids[], ref_similarity, style_conformance }
사용법

👤 김짠부가 1안 선택. 우승 제목은 학습으로 환류돼 다음 제목의 문구 감각을 끌어올린다(문구↑).

썸네일 (thumbnail_maker)

동작 구조

📖 선택 제목 + 말투(L3) + 썸네일 스타일(L3) + 우승 썸네일 문구(few-shot).

🤖 AI 1회 또는 로컬 생성. 🛡️ 주제 핵심 키워드 누락 시 경고(topic_missing).

출력

썸네일 3안 — 메인 문구·박스 문구·배치. 💾 stage_proposals.

{ thumbnail_main[], thumbnail_boxes[], thumbnail_layout, reason }
사용법

👤 김짠부가 선택. 우승 썸네일 문구는 다음 제작에 few-shot으로 다시 쓰인다.

③ 구성어떻게 풀어낼까

크루: 구다리 · roleId structurer

동작 구조

📖 주제·제목 + 구성 인사이트(L3, 과거 성공 구조) + 쉬운설명 톤(L3) + 구성 스타일(L3, 실제 목차 few-shot).

🤖 AI 1회 호출.

출력

구성 2~3안 — 전개 전략 한 문장 + 섹션 목차. 💾 stage_proposals.

{ approach, outline:[{section,note}], reason }
사용법

👤 김짠부가 선택. 선택된 구성이 다음 단계 리서치가 '무엇을 검증할지'의 기준이 된다.

④ 리서치사실을 교차검증한다

크루: 셜록 셀 · 5단계 fan-out → join (researchCell.ts)

한 명의 AI가 아니라 5개 역할이 분업한다. 검증 → 정리 → 자료 생성 → 반론 순서로 흐르고, 애매한 사실은 항상 👤 사람 검수로 올라간다.

1
셜록 팀장 · sherlock_lead

🤖 구성을 받아 '검증해야 할 것'을 분해한다. claims(사실 주장)과 concepts(시청자가 어려워할 개념)를 뽑고, 금융·통계 주장엔 강검증 플래그(is_financial)를 단다.

2
팩트검증가 · fact_verifier ×N (병렬)

🤖 claim마다 웹검색 후 검증 — 출처 계층·인용·독립 출처 수·신선도까지 판정. 금융 claim은 공식 도메인만 검색. 🛡️ 검색결과 인젝션 방어. 검증 실패 시 강등(could_not_verify)해 사람 검수로 보낸다.

3
교차정리 · reconcile (⚙️ AI 0회)

코드 로직으로 7무결성 가드·삼각검증(독립 출처 ≥ 2)·인용 확인·상충 탐지를 돌린다. 미검증·논쟁 사실은 👤 사람 검수로 escalation.

4
셈이 · numbers + 유이 · analogist (병렬)

🤖 검증된 사실에만 근거해 숫자 예시(셈이)와 일상 비유(유이)를 만든다. ⚙️ 검산·왜곡 여부는 코드로 한 번 더 확인.

5
반론 · critic

🤖 확증편향을 차단한다. 빠뜨린 주장·반증을 찾아 균형을 맞춘다.

출력

💾 research_facts(검증 사실, 사람 승인 대기) · explanation_assets(숫자·비유).

사용법

👤 김짠부가 사실을 검수·승인. 승인된 사실만 대본에서 단정할 수 있다.

원칙

팩트체크 100% 보장은 불가하다고 인정 — 출처 명시 + 교차검증 + 사람 최종확인으로 위험을 낮춘다.

⑤ 대본김짠부 말투로 쓴다

크루: 짠펜 · roleId scribe

동작 구조

📖 승인된 사실 + 설명 자산(숫자·비유) + 말투(L3).

🤖 AI 1회 호출(웹 차단). 미검증 사실엔 '단정 금지' 라벨. 🛡️ 표절 가드(코퍼스 shingle)·신선도 게이트(오래된 사실이면 리서치 재진입).

출력

대본 조각 + 출처 연결(lineage) — 문장마다 어떤 사실·자산에서 왔는지 기록. 💾 script_segments (+ segment↔fact/asset).

{ segments:[{ ord, text, used_fact_idxs[], used_asset_idxs[] }] }
사용법

👤 김짠부가 검수. 영상 발행 뒤 성과가 학습 루프의 재료가 된다.

학습성과로 똑똑해지는 루프

발행 → 성과 → 회고 → 지식 승격이 한 바퀴다. 학습 재료 = (처음 제안 ↔ 김짠부 최종본의 차이) × 성과.

  • 성과 수집 — performance_metrics: 24h 클릭률·조회수·A/B (⚠️ 현재 수동 입력).
  • 추출 에이전트 — 말투추출(tone) · 제목 스타일(title) · 썸네일 스타일(style) · 구성 스타일(structure) · 회고(retrospectivist) · 교정차이분석(correction_diff).
  • L3 지식 — tone_profile · style_profiles(제목·썸네일·구성) · insights(승인) · correction.
  • 주입 — 사람 활성화 게이트를 통과한 지식만 각 단계 프롬프트에 다시 들어간다 (말투→대본, 문구→제목·썸네일, 구성→구성, 인사이트→주제).
자동 생성 지식을 곧바로 쓰지 않고 사람이 활성화(active)해야 반영된다 — 검증 안 된 학습이 품질을 떨어뜨리는 것을 막기 위해서다.

기술전체 기술 구조 한눈에

데이터 입력 3경로 → 5단계 파이프라인(읽기·AI호출·가드·저장) → 성과·학습 환류까지를 한 장에 담았다.

개발자용 — 각 단계의 roleId · 읽는 입력(L1/L3) · AI 호출 · 가드 · 저장 테이블. 리서치는 5개 서브에이전트로 펼쳐진다.

스택: Next.js + TypeScript + Supabase + Tailwind v4 + shadcn/ui + Vercel + Inngest(durable 파이프라인). 모든 AI 호출은 src/llm/callLLM() 한 경로로만 — 개발은 fixtures 리플레이로 과금 0, 운영은 Anthropic API.
