# 썸네일 두 메인문구 어미 대비 (thumbnail-main-ending-variety)

작성일: 2026-07-03

## 배경 / 문제

김짠부 채널은 썸네일 상단/하단 메인문구를 **같은 어투로 쓰지 않는다** — 하나가 `~요`로 끝나면
다른 하나는 `~요`로 끝내지 않고 어미를 대비시킨다. 그런데 현재 생성물은 `thumbnail_main` 2개가
**둘 다 `~요`로 끝나** 단조롭고 어색하다.

## 진단 (코드로 확인)

- 썸네일 한 장 = `thumbnail_main` 정확히 2개(상단/하단 큰 문구) + `thumbnail_boxes` 2개
  (schema.ts). 문제는 그 두 main의 어미가 둘 다 `~요`인 것.
- 기존 어투 규칙은 **전부 프롬프트로만** 걸려 있고 코드 가드 없음. 좋은 예시
  (`["통장에 돈 묵히면 손해","파킹통장이 정답"]`)는 둘 다 명사 종결로 이미 대비돼 있으나,
  "둘의 어미를 다르게"가 **명시돼 있지 않아** 모델이 `~요~요`로 드리프트한다.
- ★기존 "명령·권유는 존댓말 종결(~하세요/~됩니다)" 규칙과 충돌 금지 — 그래서 규칙은 "`~요` 전면 금지"가
  아니라 "**둘 다 `~요`로 끝내지 마라**(하나는 체언·명사/감탄 종결로 대비)"이다.

## 결정 (추천안 = A + B-로깅)

- **A. 프롬프트 규칙 (핵심)**: 두 메인문구 어미를 대비시키는 규칙 + `✗ 둘 다 ~요` 반례를
  `THUMBNAIL_MAKER_SYSTEM`에 **덧붙인다**(기존 문장 재작성 금지).
- **B. 관측 로깅 (보강·강제 아님)**: 생성 후 두 main이 **둘 다 `~요`**인지 순수 헬퍼로 검출하고,
  `thumbnailToCandidates`에서 위반 시 `console.warn`으로 관측한다. **강제 거부·재생성 없음** —
  프롬프트 규칙이 얼마나 듣는지 지켜본 뒤 필요하면 강화(정답셔플과 달리 어미는 코드로 의미적
  자동수정 불가하므로 A가 주 레버).

## 설계

### 순수 헬퍼 `src/agents/thumbnail_maker/mainEndings.ts` (topicMissing.ts 미러 위치)

```ts
/** 두 메인문구가 둘 다 '요'로 끝나는가(어미 단조 검출). 후행 공백·문장부호(?!.…~) 제거 후 판정.
 *   main이 2개 미만이거나 빈 문자열이면 false(방어). 순수·throw 0. */
export function bothMainEndWithYo(main: string[]): boolean;
```

- 정규화: 각 문구 `trimEnd` + 후행 `?!.…~ ` 류 제거 → 마지막 글자가 `요`인지.
- 정확히 2개 중 **둘 다** `요` 종결일 때만 true. 하나만/둘 다 아님 → false.

### 관측 로깅 (stage.ts `thumbnailToCandidates`)

- 각 후보 매핑 시 `if (bothMainEndWithYo(c.thumbnail_main)) console.warn(...)` 한 줄
  (idx·문구 포함, 강제 거부 아님). `topic_missing`처럼 **생성 후 부착 로직 옆**에 둔다 — promptHash 무관.
- payload 스키마/타입은 **불변**(관측은 로그로만, 표시 필드 추가 없음 — "로깅만" 결정에 정렬).

### 프롬프트 규칙 (schema.ts `THUMBNAIL_MAKER_SYSTEM`)

기존 어투 항목(★어투 …) **바로 뒤에 한 줄 덧붙임**:
- "★어미 대비: `thumbnail_main` 두 문구의 어미를 다르게 쓴다 — **둘 다 `~요`로 끝내지 마라**.
  하나가 존댓말 종결(`~하세요/~됩니다/~요`)이면 다른 하나는 체언·명사 종결(예: `정답/손해/필수 시청`)이나
  감탄으로 대비를 준다. 예 ✗ `['지금 사야 해요','손해 봐요']` → ✓ `['지금 사야 해요','결국 손해']`.
  (김짠부는 상·하단 어미를 다르게 쓴다.)"
- 기존 존댓말 종결·정중-탐문 금지 규칙과 충돌하지 않는다(하나는 여전히 존댓말 종결 가능).

## 테스트 `tests/thumbnailMainEndings.test.ts`

- 둘 다 `요` 종결 → true (`["지금 사야 해요","손해 봐요"]`).
- 하나만 `요` → false (`["지금 사야 해요","결국 손해"]`).
- 둘 다 `요` 아님 → false (`["통장에 돈 묵히면 손해","파킹통장이 정답"]`).
- 후행 문장부호 방어: `["살까요?","팔까요?"]` → true (`?` 제거 후 `요`).
- main 2개 미만·빈 문자열 → false(방어).

## 불변식 / 하위호환

- 관측은 로그만 — payload·스키마·UI·계약 불변. 후보를 거부·재생성하지 않는다.
- `THUMBNAIL_MAKER_SYSTEM` 변경 → promptHash 변경 → 기존 thumbnail fixture는 replay 미스 →
  다음 라이브 생성 시 record(기존 fixture 워크플로우와 정렬). 유닛 테스트는 순수 함수라 영향 없음.
- 마이그레이션 0·의존성 0.

## 범위 밖 (Out of scope)

- 위반 시 강제 거부·자동 재생성(B는 로깅까지만 — 지켜본 뒤 결정).
- 어미 자동 재작성(의미적이라 코드로 결정적 수정 불가).
- 로컬 생성(copy-local-gen) 스켈레톤 어미 교정 — 별개 이슈(로그로 표면화만).
- `thumbnail_boxes` 어미(이번 리포트는 main 2개 한정).
