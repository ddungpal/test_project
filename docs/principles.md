## 1. 목적과 북극성

**produce script** = 유튜브 크리에이터 **김짠부**(@zzanboo, TRUS Create)의 콘텐츠 제작 5단계(주제→썸네일/제목→구성→리서치→스크립트)를 AI 크루가 **보조·대행**하고, 학습으로 고도화되어 김짠부의 사고를 **확장**하는 제작 동료 AI.

**★ 콘텐츠 북극성 (모든 판단의 기준)**
> **"처음 듣는 사람도 편안하게 듣다 보니 이해됐다."**

이 한 문장이 합격 기준이다. 정확하지만 어려운 결과물은 **실패**다. 셜록은 개념마다 숫자 예시·쉬운 비유를 의무 발굴하고, 짠펜은 낯선 개념을 비유·예시로 **먼저** 풀고 설명한다. 미충족 시 생성 차단.

---

## 2. 제1원칙 — 김짠부는 '선택', AI는 '이유'

**김짠부는 매 단계 '선택'만 한다. 이유는 AI가 현재 방식에 근거해 설명한다.**

- AI는 항상 **N개 제안 + 각 제안의 이유**를 낸다. 단답·단일안 금지.
- 김짠부는 **고르고, 필요하면 수정**한다. 그 선택·수정은 전부 기록된다(학습 연료).
- "왜 이게 좋은가"의 설명 책임은 **AI**에 있다. 김짠부에게 분석을 떠넘기지 않는다.
- 이유는 **현재 방식·과거 성과·학습된 패턴**에 근거해야 한다. 근거 없는 권위적 단정 금지.

이 원칙이 인간-AI 분업의 헌법이다. 모든 UI·에이전트 출력은 "선택지 + 근거"의 형태를 따른다.

## 3. 데이터 3층 구조

| 층 | 성격 | 예 | 규칙 |
|---|---|---|---|
| **L1 raw** | 불변 원본 | 스크립트 import·자막·댓글·인터뷰·리서치 출처 | **UPDATE 금지**(트리거로 차단). 들어온 그대로 보존. |
| **L2 structured** | 구조화된 작업물 | contents·production_runs·제안·선택·리서치 사실·성과 | provenance(출처 FK) 필수. 어디서 왔는지 항상 추적 가능. |
| **L3 knowledge** | 승격된 지식 | insights·회고·말투 프로파일·선택 패턴·비용원장 | L2의 반복·성과에서 **승격**. 직접 입력 아님. |

**핵심 패턴 (모든 단계 공통)**
> 제안 N개 + 이유 → 선택 + 수정 기록 → 회고로 성과 연결 → 인사이트로 승격

데이터는 위로 흐른다(raw→structured→knowledge). 절대 거꾸로 원본을 덮어쓰지 않는다.

## 4. 학습 루프와 AX 전환

**학습 루프**: 선택·수정·성과가 쌓일수록 제안 품질이 올라간다.
1. 김짠부 선택/수정 → `selection_patterns`
2. 업로드 후 성과(CTR·조회·댓글 이해도 신호) → 회고
3. 반복 성과 → `insights` 승격 → 다음 제안에 반영

**모델 전략 — RAG 시작 → AX 전환 (정성적 트리거)**
- 처음엔 RAG(과거 자료 검색·few-shot)로 시작.
- 김짠부가 특정 단계에서 **"이제 됐다"** 하면 그 단계를 AX(내재화)로 전환. 채택률 수치는 보조 지표일 뿐, **정성 판단이 1차 기준**.
- **AX 우선순위: 말투 내재화 > 선제 제안 > 자율성.** 가장 먼저 내재화할 것은 "김짠부답게 말하기".

**골든셋과 코퍼스는 다르다 (회귀 방지)**
- **코퍼스**: 학습 연료. 계속 축적된다.
- **골든셋**: 채점 기준(시험지). 버전 동결 스냅샷. 학습에 쓴 것으로 시험 보면 품질 저하를 못 잡으므로 분리·버전 관리한다. (현재 `골든 v1` = 정보형 8편)

## 5. 혼동 방지 원칙 (이 프로젝트의 생명선)

금융·경제 채널이므로 **틀린·낡은 정보를 그럴듯하게 포장해 배포하는 것**이 최대 리스크다. 다음 경계를 절대 흐리지 않는다.

**(a) 사실 / 생성 / 추정의 경계**
- 검증된 사실, AI가 생성한 표현, 불확실한 추정은 **출력에서 구분**된다.
- 검증 못 한 사실은 `could_not_verify`로 표시하고 **날조하지 않는다**(우아한 실패). 짠펜은 미검증 사실로 스크립트를 못 쓴다(차단 → 셜록 재호출).

**(b) 말투 ≠ 내용**
- 학습된 "김짠부 말투"는 **표현 스타일**이지 **사실 근거가 아니다**. 말투가 그럴듯하다고 내용이 참이 되지 않는다. 말투 가드와 팩트 가드는 독립적으로 작동한다.

**(c) 최신성 (stale 방지)**
- 모든 사실에 `as_of_date` + `volatility`(static/slow/fast)→TTL. 금리·시세=fast(매번 재확인), 개념=static.
- **모델 컷오프(2026-01) 이후·시변 정보는 라이브 검색 강제.** 짠펜 freshness 게이트가 TTL 초과 사실을 차단한다.

**(d) 리서치 무결성 7대 가드** (상세 tech.md §9)
①출처 독립성(`independent_origin_count`≥2) ②인용 실재 검증(fetch+quote, 없으면 폐기) ③반론 패스(확증편향 차단) ④우아한 실패 ⑤통계 오용 검증 ⑥1차 출처+한국 현행 우선 ⑦위험 기반 사람 검수 트리아지(금융·수치·충돌·미검증·stale만 에스컬레이션).

> **컴플라이언스 경계**: 투자조언·면책 등 법적 컴플라이언스는 시스템 밖 **운영자(김짠부) 책임**. 시스템은 정확성·출처·최신성까지 보장하고, 그 한계를 정직하게 표시한다.

## 6. 비용·개발 원칙

- **개발 = $0.** `claude -p`(구독 정액·공짜 반복) + `fixtures/` 리플레이. 수정 무한 반복해도 API 과금 0. 비용 걱정 없이 빠르게 실험한다.
- **운영 = 종량.** Anthropic API(또는 Phase 2 A/B로 선택할 모델). `callLLM()` 어댑터로 dev/prod 스위치.
- **비용은 ROI로 사고한다.** 편당 비용은 콘텐츠 1편의 가치 대비 판단. 무작정 아끼기보다 "폭주 차단 + 평상시 최소화".
- **가드**: 2단 비용캡(SOFT $7 사람확인 / HARD $10 중단), `max_rework=2`. 모든 호출은 `cost_ledger`에 기록.
