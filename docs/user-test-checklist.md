## 0. 사전 준비

- [ ] **`.env` 확인/수정** (테스트는 '진짜 인증'으로):
  - `DEV_OWNER_BYPASS=0`  ← **반드시 0** (실제 로그인 검증). 현재 1로 되어 있음.
  - `LLM_BACKEND=claude-p` · `LLM_FIXTURES=record` (실 LLM·$0. 새 주제는 새로 기록됨)
  - `SEARCH_BACKEND=tavily` (실검색; 결정적으로 하려면 `mock`)
  - `COST_SOFT_CAP_USD=7` · `COST_HARD_CAP_USD=10` (그대로)
- [ ] **owner 로그인 계정 확인**: `.env`의 `OWNER_EMAIL` / `OWNER_PASSWORD`. 임시 비번이면 본인 것으로 바꾸는 걸 권장(Supabase 대시보드 → Authentication → Users).
- [ ] **서버 2개 기동** (터미널 2개):
  - 터미널 A — Inngest: `npx inngest-cli@latest dev -u http://localhost:3001/api/inngest`
  - 터미널 B — Next: `./node_modules/.bin/next dev -p 3001`
- [ ] 브라우저로 `http://localhost:3001` 접속 가능 · Inngest 대시보드(`http://localhost:8288`)에서 함수 등록 확인
- [ ] (문제 시 참고) 에러는 `logs/errors.jsonl`에 영속 기록됨 — 테스트 중 이상하면 이 파일 먼저 확인

## 3. 촉이 — 주제 (created → topic_proposed → topic_selected)

- [ ] 촉이 단계 자동 진행, 작업 중 화면이 **자동 갱신**(LiveRefresh; 수동 새로고침 불필요)
- [ ] **주제 후보 ≥3개** 표시, 각 후보에 **근거(evidence)** 보임
- [ ] 후보 1개 선택(원하면 내용 수정 + 선택 이유 입력) → **선택 반영**(topic_selected)
- [ ] AI가 제시한 후보/근거가 김짠부 도메인(재테크)에 맞는지 눈으로 확인

- [ ] 김짠부 말투의 제목인지 확인(예: "찐짠이들", "~하면 손해")
- [ ] 1안 선택 → titles_selected

## 6. 셜록 — 리서치 트리아지 (researching → research_ready → research_approved)

- [ ] 리서치 시작 → **서브진행바**가 단계 표시: `검증 범위(셜록) → 팩트 검증 → 교차 정리 → 숫자·비유(셈이∥유이) → 반론`
- [ ] research_ready 후 **검수 진입**
- [ ] **에스컬레이션된(위험) fact만 검수 대상**으로 뜨고, 자동 통과 건수도 표시
- [ ] fact 카드의 **무결성 뱃지** 확인: 검증상태 · 금융 표시 · 출처 등급(tier) · 독립출처 수 · 인용 확인
- [ ] **출처 링크 클릭** → 정상 이동(http/https만 링크, 그 외 텍스트 폴백 — 안전 스킴)
- [ ] 승인/반려 토글 후 승인 → **research_approved**
- [ ] (이번 작업 확인 포인트) 셈이의 숫자/유이의 비유가 **검증된 사실 뒤에** 생성됐는지 흐름상 문제없음

## 7. 짠펜 — 대본 검수 (scripting → script_ready → approved)

- [ ] **단락(segment)** 표시 + **lineage 칩**(각 단락이 쓴 fact/숫자/비유 연결)
- [ ] 김짠부 말투("짠하! 안녕하세요…") 반영 확인
- [ ] **표절 포함도** 표시(0.6 미만)
