김짠부의 사고를
확장하는 AI 크루

유튜브 크리에이터 김짠부의 콘텐츠 제작 5단계를 AI 크루가 보조·대행하고, 학습으로 고도화된다. 김짠부는 매 단계 '선택'만 하고, 이유는 AI가 현재 방식에 근거해 설명한다.

★ 콘텐츠 북극성 — "처음 듣는 사람도 편안하게 듣다 보니 이해됐다."

How it works

제작 5단계 파이프라인
STEP 1
주제
촉이
댓글·트렌드·지표에서 주제 후보 N개 + 근거 제안
STEP 2
썸네일·제목
훅이
제목·썸네일 문구 3안(A/B/C) + TRUS 디자인 실물
STEP 3

STEP 4
리서치

사실 검증 + 숫자예시·비유 발굴 (병렬→조인→반론)
STEP 5
스크립트
짠펜
말투·쉬운설명·근거 게이트 통과한 대본
총괄 = 반장(PD·오케스트레이터) · 모든 단계를 Inngest durable 파이프라인으로 조율 · 각 단계 끝에서 김짠부가 선택
반장 상태머신: created → topic_proposed 선택 → titles_proposed 선택 → structure_proposed 선택 → researching(셜록 셀) → research_ready → research_review 트리아지 승인 → research_approved → scripting → script_ready → script_review 선택 → approved → published

The crew

크루 — 각 에이전트의 역할
반장총괄 PDorchestrator
전체 파이프라인을 조율하는 오케스트레이터. 단계 전이·rework 재호출·비용 가드·사람 게이트(선택 대기)·중단(kill switch)을 관리.
상태머신 · fan-out/join · max_rework=2 · 2단 비용캡
촉이주제topic_scout
댓글 마이닝·트렌드·경쟁채널·경제캘린더·지표에서 주제 후보를 발굴. 각 후보에 "왜"의 근거를 단다.
IN 댓글·metrics → OUT 후보 ≥3 (+evidence)
훅이썸네일·제목hook_maker
제목·썸네일 문구를 3안(A/B/C)으로. 레퍼런스 CTR·말투·TRUS 디자인 스펙 기반. 업로드 후 A/B 성과를 환류 학습.
IN 주제·CTR·tone → OUT 제목3 + 썸네일3
구다리구성structurer
콘텐츠 구조를 짠다. 섹션별 목표·왜 이 순서인지·이해 흐름(맥락·불안완화·오개념 선제제거)을 담은 outline 산출.
IN 주제·구조비율·retention → OUT outline
셜록리서치 팀장sherlock_lead
리서치 셀의 오케스트레이터. outline을 검증 대상(claims)과 설명 대상(concepts)으로 스코핑 → 전문가 팬아웃 → 리콘실 → 반론. 아래 4인을 지휘.
IN outline → OUT scope {concepts[], claims[]}
짠펜스크립트scribe
최종 대본 작성. 낯선 개념은 숫자예시·비유를 먼저 제시. 말투·쉬운설명·freshness·표절 가드를 모두 통과해야 함. 미충족 시 생성 차단 → 셜록 재호출.
IN outline·facts·assets·tone → OUT script_segments
🔍 셜록 리서치 셀 — 병렬 전문가 (팬아웃 → 조인 → 반론)
스코핑 후, 세 전문가가 동시에 파고들고 → 정합성을 맞춘 뒤 → 반론으로 흔든다. 자세한 흐름은 research-flow.html 참고.

🎯제1원칙 — 선택 vs 이유

  • 김짠부는 매 단계 '선택'만, 이유는 AI가 현재 방식 근거로 설명
  • 각 제안에 근거(evidence) 링크 필수
  • 선택 + 수정 이유를 한 줄로 캡처해 학습

✅팩트체크 · 리서치 무결성

  • 출처 명시 + 교차검증(독립 ≥2) + 사람 최종확인
  • 인용 실재 검증(quote_excerpt), 없으면 폐기
  • 금융 claim = 1차 출처 강제 + 사람 승인 필수
  • 검색 실패 = could_not_verify → 짠펜이 사용 차단(날조 금지)
  • 100% 보장 불가 인정 — 면책·투자조언 경계는 운영자 책임

⏱최신성 (오늘=2026-06-18)

  • volatility(static/slow/fast) → TTL 차등(금리·시세=매번 재확인)
  • 모델 컷오프(2026-01) 이후 시변 정보는 라이브 검색 강제
  • 짠펜 freshness 게이트: stale면 차단 → 재확인

🛡인젝션 방어 · lineage

  • 신뢰불가 입력(댓글·웹·자막)은 <<UNTRUSTED_DATA>> 격리
  • 에이전트별 tool 화이트리스트 · 출력 schema 강제
  • 대본 문장 → 근거 → 출처 FK 조인 역추적(배열/JSON 금지)
  • 미검증 근거에 의존한 문장 = 발행 차단
