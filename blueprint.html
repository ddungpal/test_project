05팩트 엔진 & 학습 건강도

정확도는 이 시스템의 생명. 셜록의 검증을 메커니즘으로 못박고, ‘정말 똑똑해지는지’를 측정한다.

🔍 셜록 팩트 엔진

claim 분해› 독립출처 N개 교차› 신뢰등급› 사람 승인

소스 신뢰등급(공식기관·언론·블로그) · 상태 = verified / conflicting / unverified · 충돌·미검증은 플래그 후 김짠부가 claim별 승인.

🟡 금융 콘텐츠 강검증 — 숫자·세법·투자정보는 별도 강검증 + 시점표기 가드. 틀리면 품질이 아니라 ‘리스크’.

06셜록 리서치 셀 & 최신성 엔진

셜록은 단독이 아니라 ‘팀장’. 실패 방식이 다른 작업을 전문가로 나눠 병렬 실행 → 빠르고 정확하게. 그리고 모든 사실엔 ‘언제 기준’인지가 붙는다.

🔍
셜록 — 리서치 팀장 · 스코핑(핵심 개념·claim 추출) → 병렬 fan-out → 리콘실(조인) 책임
🧷 팩트검증가
claim 교차검증·신뢰등급·출처 — 진실의 척추
🧮 셈이 (숫자)
구체 수치·통계·계산 발굴 + 산술·단위·시점 검증(가능하면 코드 검산) → numeric_example
🍕 유이 (비유)
일상 비유 생성(+학습된 성공비유 RAG) + 왜곡 검증 → analogy

↪︎ 리콘실(조인) — 세 결과 정합성 교차검사: 비유가 사실과 모순 안 되는지 / 숫자가 검증 통과했는지 / 충돌·미검증은 해당 전문가만 재호출(rework). 이어 반론·완전성 패스로 “무엇이 빠졌나 / 반대 근거는 / 회의론자라면?”을 점검(확증편향 차단·사고 확장). 완전 병렬이 아니라 ‘팬아웃 → 조인’.

🛡 리서치 무결성 7대 가드 — “그럴듯하지만 틀린” 리서치를 막는다
① 출처 독립성
원출처 추적 → 독립 출처 수로 카운트. 보도자료 복제기사(에코챔버) 거짓 확신 차단
② 인용 실재 검증
모든 출처 실제 fetch → 주장 담은 정확 인용문 추출. 없으면 폐기 — 환각 출처 차단
③ 반론 패스
반대근거·누락 점검 → 반대증거 검색. 확증편향 차단(사고 확장과 직결)
④ 우아한 실패
못 찾으면 could_not_verify 표기·날조 금지 → 짠펜이 미검증 사실 사용 차단
⑤ 통계 오용 검증 (셈이)
명목/실질·세전후·평균의 함정·체리피킹·복리 가정 — 산술 맞아도 오도 여부까지
⑥ 1차 출처 + 한국 현행
핵심 claim은 공식 1차 출처(국세청·금융위·한국은행·통계청·법령), 한국 현행 기준 라이브
⑦ 위험기반 트리아지
금융·수치·충돌·미검증·stale만 사람 승인, 나머지 자동 통과 — 김짠부 병목 방지
⏱ 최신성 엔진 — 오늘 기준 2026-06-17
정보는 시간이 지나면 변한다. 모든 사실은 ‘언제 기준’인지를 달고, 변동성에 따라 재확인 주기(TTL)를 갖는다.
변동성 → TTL금리·시세·세법한도 = 빠름(짧은 TTL, 매번 재확인) · 개념 정의·역사적 사실 = 정적(장기/없음)
오늘 기준 라이브모델 지식은 2026-01 컷오프 → 그 이후·시변 정보는 모델 기억 금지, 라이브 검색 필수. as_of_date·source_published_at 기록
검색은 최신 우선(날짜정렬·공식 최신) · 캐시는 TTL 인지(빠른 변동은 캐시 안 함) · 오래된 시변 사실은 짠펜 freshness 게이트에서 차단 → 재확인
